

## Directory structure of `.plan/`

```
.
├── phase-1/            # These are all identical
├── phase-2/            #
├── phase-3/            #
│   ├── REVIEW.md       # This is the plan for the `reviewer` detailing what to review for each checkpoint in this phase of the project
│   ├── WORKER.md       # This is the plan for the `junior-dev` detailing what work to do for each checkpoint in this phase of the project
│   ├── DOCUMENT.md     # This is the plan for the `tech-writer` detailing what documentation is due for this phase of the project
│   └── README.md       # This file. Describes the purpose and layout of the "phase-X" directory and its contents; Also describes the focus of this phase of work
├── phase-N/
├── index.md
└── README.md
```
